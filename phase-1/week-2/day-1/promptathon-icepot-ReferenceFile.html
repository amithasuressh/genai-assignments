ICEPOT Framework · Prompt Engineering

Promptathon

Transform weak, vague prompts into powerful, structured prompts using the ICEPOT Framework. Pick any 3 scenarios from the 7 below and complete the challenge.

🎯 Your Challenge

You will be given 7 scenarios, each with a weak prompt. Choose any 3 that interest you. For each scenario, identify what is missing, rebuild the prompt using ICEPOT, run both prompts in an AI tool, and observe the difference in output quality.

Steps to follow
1
Read all 7 scenarios below and choose any 3 that interest you.
2
For each chosen scenario, read the weak prompt carefully and write down what information is missing from it.
3
Fill in each ICEPOT component in the template to build your improved, structured prompt.
4
Run the weak prompt first in ChatGPT, Gemini, or Claude or qwen (any one) and save the output.
5
Now run your rewritten ICEPOT prompt in the same AI tool and save that output.
6
Compare both outputs side by side and write 3 observations on what improved and why.
7
Repeat Steps 2 to 6 for your remaining 2 chosen scenarios.
7 Scenarios — Pick any 3
Scenario 01
🔍 Search Functionality
✗
"Test the search feature."
What's missing
No product type (e-commerce, hospital), no filter/sort behavior, no empty/invalid input scenarios, no performance angle.
Scenario 02
📎 File Upload
✗
"Generate test cases for file upload."
What's missing
No file types, no size limits, no success/failure criteria, no security angle (malicious file upload).
Scenario 03
🛒 Shopping Cart
✗
"Test the cart functionality."
What's missing
No add/remove/update actions, no price calculation validation, no coupon/discount scenarios, no out-of-stock handling.
Scenario 04
🔐 Role-Based Access Control
✗
"Write test cases for user roles."
What's missing
No role types (Admin, Manager, Viewer), no permission matrix, no unauthorized access scenarios, no app domain.
Scenario 05
🌐 API Testing
✗
"Write API test cases for user registration."
What's missing
No HTTP method, no request/response structure, no status codes to validate, no tool context (Postman/RestAssured).
Scenario 06
📱 Mobile App — OTP Login
✗
"Test OTP login on mobile."
What's missing
No platform (Android/iOS), no OTP expiry time, no resend behavior, no network failure scenario, no Appium/device context.
Scenario 07
📊 Dashboard / Report Generation
✗
"Test the reports section."
What's missing
No report type (PDF/Excel/CSV), no date range filters, no data accuracy validation, no permission-based visibility.
